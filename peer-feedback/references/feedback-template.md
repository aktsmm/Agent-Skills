# ピアフィードバック 6項目テンプレート

半期ピアフィードバックの記載テンプレートと記載例。
**基本方針: 全項目ポジティブ、「再考」セクションも成長余地として前向きに表現する。プロモーション（昇進・昇格）に役立つ具体的な成果・行動・影響を書く。**

---

## 構造

```
# {対象者名} へのフィードバック

## 継続してほしいこと

### 私が思うあなたのとても良いところと継続してほしいこと
### あなたの強みをより活かすための私からの提案

## 再考してほしいこと（任意）

### 私が思う考え直してはどうかと思うこと
### 私が思う別の方法で実施してはどうかと思う具体的な例

## 追考

### 私があなたと協業していて最も価値を感じることは
### その他私からのフィードバック
```

> 「再考してほしいこと」は、記載できる具体例がない場合はセクションごと省略してよい。

---

## 各項目の書き方ガイド

### 継続してほしいこと: 良いところ

**目的**: その人の強み・行動・成果を具体的に伝え、継続を促す

**書き方のポイント**:

- 具体的なプロジェクト名や出来事を添える
- 「何をした」だけでなく「それがどう良かったか（インパクト）」まで書く
- チームや周囲への影響範囲を言語化する

**記載例**:

```
〇〇プロジェクトでは、複数の利害関係者が絡む要件定義をリードし、短期間で合意形成を実現していただきました。
技術的な深さと相手の立場への配慮を両立した進め方は、チーム全体の進捗を加速させる大きな推進力でした。
この「調整力 × 技術力」の組み合わせはあなたの際立った強みです。ぜひ今後の大規模案件でも発揮し続けてください。
```

---

### 継続してほしいこと: 強みをより活かすための提案

**目的**: 強みをさらに広げ、影響範囲を拡大するための提案をする

**書き方のポイント**:

- 「今よりもっとできる」という期待として表現する
- 具体的なアクション・場面を提案する
- 相手の成長・キャリアに繋げるトーンで

**記載例**:

```
あなたのファシリテーション力は社内でも高く評価されています。
今後はチーム内にとどまらず、部門横断の設計レビューや社外との技術共有の場でも積極的に発揮していただけると、
組織全体への影響力がさらに広がると思います。
```

---

### 再考してほしいこと: 考え直してはどうか（任意）

**目的**: 改善余地を、相手の成長・次のステージへの期待として伝える

**書き方のポイント**:

- 「批判」ではなく「より大きな成果を出すための気づき」として書く
- 「〜してほしくない」より「〜するとさらに良くなる」の表現を使う
- 相手のプロモーションを念頭に「次のレベルで求められること」として位置付ける

**ポジティブ re-frame 変換例**:

| 事実（そのままは書かない） | re-frame 後の表現                                                                        |
| -------------------------- | ---------------------------------------------------------------------------------------- |
| 報告が遅い                 | 「成果の可視化をより早めることで、周囲からのサポートを得やすくなります」                 |
| 一人で抱え込む             | 「チームへの早期共有を習慣化すると、影響力がさらに広がります」                           |
| 説明がわかりにくい         | 「非技術者にも伝わる言語化を意識すると、提案の説得力が増します」                         |
| 消極的に見える             | 「知見を積極的に発信することで、より多くの人がその価値を知ることができます」             |
| 意見の衝突があった         | 「率直に議論し合い、そのたびにより良い判断に着地できた。正面からぶつかっても壊れない信頼があるからこそ成果物の質が上がった」 |

**記載例**:

```
現在の業務領域では確かな成果を出し続けていますが、次のステージを見据えると、
成果やプロセスを早めに周囲に見せることが重要になってきます。
あなたの取り組みはとても価値が高いので、その価値が組織全体に伝わるような
情報共有のリズムを作っていただけると、さらなる評価につながると期待しています。
```

---

### 再考してほしいこと: 別の方法の具体的な例（任意）

**目的**: 改善提案を「具体的なアクション」として示す

**書き方のポイント**:

- 「こうすればよかった」の批判ではなく「こうすると良くなる」の提案
- 一つだけ具体的な場面・方法を挙げる

**記載例**:

```
例えば、大型案件の中間報告をチームの週次会議に組み込む形を取るだけで、
進捗の可視性が上がり、周囲が支援しやすい環境になります。
実際に似たシナリオで効果が出ているため、ぜひ試してみてください。
```

---

### 追考: 協業していて最も価値を感じること

**目的**: 相手の存在・関与から生まれた具体的な価値を伝える

**書き方のポイント**:

- 「あなたがいてくれたから〇〇ができた」という具体的な体験を書く
- 技術・スキルだけでなく、人柄・姿勢・チームへの影響も書いてよい
- プロモーション評価で使える「他者への影響・組織貢献」を意識する

**記載例**:

```
〇〇案件の佳境で、私が技術的な判断に迷っていた際に、あなたが即座にレビューを引き受けてくれたことが
とても助かりました。あなたの判断の速さと、常に相手の状況を考えてくれる姿勢は、
チームの心理的安全性を高めるとても大きな力だと感じています。
```

---

### 追考: その他フィードバック

**目的**: 上記に収まらない観察・期待・応援メッセージを自由に書く

**書き方のポイント**:

- 人柄・成長・将来への期待などを書いてよい
- 短くても心のこもった一言で締めくくると印象的

**記載例**:

```
半期を通じて、あなたの仕事への真摯な姿勢と周囲への配慮を間近で見てきました。
技術力だけでなく、人を動かす力も着実に高まっていると感じます。
次のステップでのさらなる活躍を楽しみにしています。
```

## ライティングチェックリスト（出力前確認）

- [ ] マークダウン装飾（`**` 等）を使っていないか（フォームコピペで崩れるため禁止）
- [ ] 会社のコアバリューやミッションと紐付けた表現が自然に含まれているか
- [ ] 具体的なプロジェクト名・出来事が含まれているか
- [ ] 「良かった」だけでなくインパクト（影響・成果）まで書いているか
- [ ] 「再考」セクションはポジティブ re-frame されているか（批判になっていないか）
- [ ] プロモーションを後押しする成果・影響範囲の記述があるか
- [ ] 相手が読んで前向きな気持ちになれるトーンか
- [ ] 事実に基づいているか（workIQ 収集結果または記憶に根拠があるか）
