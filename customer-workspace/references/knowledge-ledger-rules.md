# Knowledge Ledger Rules

Use `_knowledge/` only when the user asks to extract, generalize, or preserve reusable learnings from meetings, incidents, or research.

- Start with `_knowledge/general.md`; split categories only after entries accumulate enough to justify it.
- Store compact patterns, gotchas, decision criteria, and validation checks. Put long analysis in `research-reports/`.
- Remove customer and person names, ticket IDs, local paths, and unpublished or confidential details.
- Mark Microsoft product, pricing, support, and roadmap claims as requiring official confirmation unless a checked source URL is included.
- Link to source meeting notes or reports instead of copying long transcripts.

Use `research-reports/` for generated research notes, comparison memos, report drafts, and demo specifications. Keep the workspace root reserved for entry files and workspace controls.
