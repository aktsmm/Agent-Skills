# 質問事項抽出（会議ログ → Questions）

## 概要

会議ログ（議事録 / Teams AI メモ / 手書きメモ）から質問・確認事項・宿題を抽出し、指定した保存モードで書き出す。

---

## 保存モード

ユーザーが明示しなければ質問する:

| モード | 保存先 | 用途 |
| --- | --- | --- |
| **per-meeting** | `{日付}/{日付}_質問事項.md` | 会議ごとに分けて管理したい |
| **accumulate** | `_questions/{YYYY-MM}.md` | 月単位で一括蓄積したい |

---

## 処理フロー

### Step 1: 保存モードの決定

1. ユーザーが「会議ごと」「分けて」→ `per-meeting`
2. ユーザーが「一つに」「まとめて」「蓄積」→ `accumulate`
3. 明示なし → 質問して確認

### Step 2: 日付の決定

- 議事録ファイルが開いている → ファイル名から日付抽出
- 日付が見つからない → 今日の日付を使用

### Step 3: 質問事項の抽出

以下を「質問事項」として抽出:

| パターン | 例 |
| --- | --- |
| 疑問文（？で終わる） | 「〜は可能ですか？」 |
| 確認依頼 | 「〜を確認してください」「〜の確認をお願い」 |
| 宿題・持ち帰り | 「持ち帰り」「宿題」「次回までに」 |
| 要回答 | 「回答をお願い」「ご教示ください」 |
| TODO / Action Item | 「TODO:」「AI:」「Action:」 |
| フォローアップ タスク | Teams AI の「フォローアップ タスク」セクション |

### Step 4: 分類

抽出した項目を以下に分類:

| 分類 | 説明 |
| --- | --- |
| 📌 未回答 | まだ回答されていない質問 |
| ✅ 回答済み | 同じ会議内で回答が出た質問 |
| 🔄 持ち帰り | 次回以降に回答予定 |

### Step 4.5: 回答を埋めるときのファクトチェック（技術仕様・課金・モデル・リリース状況に関する質問のみ必須）

Microsoft / Azure / Microsoft 365 / Copilot の **仕様・制限・課金・モデル名・GA / Preview / Retirement** に関する主張は、会議中の言質や記憶をそのまま回答に使わず、`microsoftdocs` MCP または `Fact Checker` subagent で公式 Docs / release notes に裏取りしてから書く。

- モデル名を固定して言わない（Microsoft 側で予告なく差し替える契約条項がある）。「現時点で X を underlying model として使用」と表現する。
- **公式 SLA / 限値がない数値**（「インデックス更新は X 週間」「コストは Y 円」等）は **体感・公式保証なし** と明記する。
- 出典 URL（`learn.microsoft.com/ja-jp/...` を優先）を回答本文や末尾に含める。ロケール非依存 URL が返ってきた場合は ja-jp に置換してから貼る。

### Step 5: 保存

#### per-meeting モード

```markdown
# 質問事項: {日付}

## 📌 未回答

- [ ] 質問内容（担当: xxx）

## ✅ 回答済み

- [x] 質問内容 → 回答内容

## 🔄 持ち帰り

- [ ] 質問内容（担当: xxx、期限: 次回）
```

#### accumulate モード

`_questions/{YYYY-MM}.md` の末尾に追記:

```markdown
---

### {日付} 会議

- [ ] 質問内容（担当: xxx）
- [x] 質問内容 → 回答内容
```

### Step 6: 確認メッセージ

```
📋 質問事項を抽出しました: {件数}件（未回答: {n}件、回答済み: {n}件、持ち帰り: {n}件）
保存先: {ファイルパス}
```

---

## 使用例

```text
この議事録から質問事項を抽出して、会議ごとに分けて保存して
```

```text
今日の会議メモから質問を抜き出して、一つのファイルにまとめて
```
